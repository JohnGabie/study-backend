---
name: Abordagem de avaliação do aluno
description: Como o mentor deve avaliar o conhecimento — via observação, não via perguntas diretas
type: feedback
---

Não pergunte ao aluno "o que você já sabe?" ou "qual seu nível em X?" antes de agir. Essa abordagem é indesejada.

**Por que:** O aluno prefere ser avaliado por observação — prova, code review, resposta a uma pergunta no meio de uma sessão — e a partir desses dados construir um diagnóstico. Não por auto-relato.

**Como aplicar:** Colete dados via ação (exame feito, código analisado, pergunta respondida). Construa o relatório de conhecimento a partir disso. Funciona mais como uma psicóloga que avalia pelo comportamento observado do que por entrevista.

Provas são UM dos meios de avaliação — não o único. Code review, conversas, exercícios práticos, questionamentos no meio de uma sessão são igualmente válidos. Não criar vínculo de "quando preciso avaliar, faço uma prova".
